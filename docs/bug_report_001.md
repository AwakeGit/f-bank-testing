# BUG-001 — Отрицательная сумма перевода принимается без валидации

| Поле | Значение |
|------|----------|
| **ID** | BUG-001 |
| **Заголовок** | Кнопка «Перевести» активна при отрицательной сумме перевода |
| **Компонент** | Форма перевода — поле «Сумма перевода» |
| **Severity** | Critical |
| **Priority** | High |
| **Статус** | Open |
| **Тест-кейс** | TC-004 |

---

## Описание

В поле «Сумма перевода» можно ввести отрицательное значение. Приложение принимает его как корректное: вычисляет отрицательную комиссию и отображает кнопку «Перевести», что позволяет инициировать перевод на отрицательную сумму.

## Шаги воспроизведения

1. Открыть `http://localhost:8000/?balance=30000&reserved=20001`
2. Нажать на карточку «Рубли»
3. Ввести номер карты: `1111 1111 1111 1111`
4. В поле «Сумма перевода» ввести: `-100`

## Ожидаемый результат

Приложение отклоняет ввод или отображает сообщение об ошибке валидации.  
Кнопка «Перевести» не отображается.

## Фактический результат

- Поле принимает значение `-100`
- Отображается: «Комиссия: -10 ₽»
- Кнопка «Перевести» отображается и доступна для нажатия

## Скриншот / доказательство

```
Поле суммы: -100
Комиссия:   -10 ₽
[Перевести]          ← кнопка активна
```

## Ожидаемое поведение

Сумма перевода не может быть отрицательной. Поле должно принимать только положительные числа. При вводе отрицательного значения кнопка «Перевести» должна скрываться.
