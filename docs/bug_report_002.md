# BUG-002 — Неверный расчёт комиссии приводит к разрешению запрещённых переводов

| Поле | Значение |
|------|----------|
| **ID** | BUG-002 |
| **Заголовок** | Комиссия округляется до десятков вместо единиц, допуская переводы сверх доступного остатка |
| **Компонент** | Форма перевода — расчёт комиссии и проверка лимита |
| **Severity** | Critical |
| **Priority** | High |
| **Статус** | Open |
| **Тест-кейс** | TC-005 |

---

## Описание

Согласно спецификации, комиссия рассчитывается по формуле:

```
комиссия = floor(сумма × 0.1)
```

Фактически приложение использует формулу:

```
комиссия = floor(сумма / 100) × 10
```

Разница: правильная формула округляет до **единиц**, фактическая — до **десятков**.  
Из-за этого для сумм, не кратных 100, комиссия занижается (до 9 ₽), и переводы, которые должны блокироваться по лимиту, проходят проверку.

## Шаги воспроизведения

1. Открыть `http://localhost:8000/?balance=30000&reserved=20001`  
   (доступная сумма: 30000 − 20001 = **9999 ₽**)
2. Нажать на карточку «Рубли»
3. Ввести номер карты: `1111 1111 1111 1111`
4. В поле «Сумма перевода» ввести: `9091`

## Ожидаемый результат

```
Комиссия: floor(9091 × 0.1) = floor(909.1) = 909 ₽
Итого:    9091 + 909 = 10000 > 9999
→ Отображается: «Недостаточно средств на счете»
```

## Фактический результат

```
Комиссия: floor(9091 / 100) × 10 = floor(90.91) × 10 = 90 × 10 = 900 ₽
Итого:    9091 + 900 = 9991 ≤ 9999
→ Кнопка «Перевести» активна
```

## Влияние

Суммы переводов от 9091 до 9099 ₽ ошибочно разрешаются, хотя должны быть заблокированы. Аналогичная проблема возникает для любой суммы с дробной частью при делении на 10.
